# Route engineering risks with Anthropic multi-agents and Slack alerts

This document provides a comprehensive technical breakdown of the n8n workflow: **AI Engineering Governance System with Multi-Agent Safety and Risk Routing**.

### 1. Workflow Overview
This workflow automates engineering governance by deploying a multi-agent AI system (powered by Anthropic's Claude) to validate designs, verify compliance, optimize safety, and predict maintenance needs. It is designed for high-stakes industries (Aerospace, Manufacturing, Energy) where manual review is a bottleneck.

**Logical Lifecycle:**
*   **1.1 Data Ingestion:** Triggers every 6 hours to fetch design specifications and live operational data from external APIs.
*   **1.2 Multi-Agent Orchestration:** Data is processed in parallel by three specialized agents (Design, Safety, and Maintenance). These agents utilize three shared sub-tools for deeper verification (Compliance, Resources, and Testing).
*   **1.3 Consolidation & Scoring:** Results are merged, and a custom JavaScript logic calculates a weighted risk score.
*   **1.4 Routing & Alerting:** Based on the calculated risk level, the system either sends critical/high-priority Slack alerts or logs medium-level issues.

### 2. Block-by-Block Analysis

#### 2.1 Configuration & Data Acquisition
This block establishes the environment variables and fetches the raw engineering data required for analysis.
*   **Nodes Involved:** `Schedule Trigger`, `Workflow Configuration`, `Fetch Design Specifications`, `Fetch Operational Data`, `Merge Engineering Data`.
*   **Details:**
    *   **Workflow Configuration (Set):** Defines placeholder URLs for APIs and sets risk thresholds (e.g., Critical = 80, High = 60).
    *   **HTTP Request Nodes:** Fetch JSON data from the configured API endpoints.
    *   **Merge Engineering Data (Set):** Combines the two API responses into a single object with a timestamp and project ID. This ensures all downstream agents have the same context.

#### 2.2 Parallel AI Analysis (Multi-Agent System)
The "brain" of the workflow, utilizing LangChain agents and Anthropic Claude 3.5 Sonnet.
*   **Nodes Involved:** `Design Validation Agent`, `Safety Optimization Agent`, `Predictive Maintenance Agent`, and their associated Models and Output Parsers.
*   **Agent Roles:**
    *   **Design Validation:** Analyzes structural integrity and material selection.
    *   **Safety Optimization:** Identifies mechanical/thermal hazards and fail-safe adequacy.
    *   **Predictive Maintenance:** Analyzes vibration/temperature patterns to predict Remaining Useful Life (RUL).
*   **Shared Tools (Sub-Agents):** 
    *   `Compliance Verification Agent Tool`: Checks against ISO/ASME/IEC standards.
    *   `Resource Coordination Agent Tool`: Manages personnel and equipment availability.
    *   `Testing Validation Agent Tool`: Designs stress and functional test protocols.
*   **Edge Cases:** Anthropic API rate limits or malformed JSON from the agents (handled by Structured Output Parsers).

#### 2.3 Consolidation & Risk Logic
This block transforms qualitative AI findings into quantitative risk metrics.
*   **Nodes Involved:** `Consolidate Analysis Results`, `Calculate Risk Scores`.
*   **Technical Role:** 
    *   **Merge Node:** Uses "Wait for all inputs" to ensure analysis from all three agents is received.
    *   **Code Node (JS):** Executes a weighted formula: **(Safety * 0.4) + (Design * 0.3) + (Maintenance * 0.3)**. It also categorizes the result into "critical", "high", "medium", or "low".

#### 2.4 Severity Routing & Notifications
The final stage determines the communication channel based on urgency.
*   **Nodes Involved:** `Route by Risk Level`, `Alert Critical Issues`, `Alert High Priority Issues`, `Log Medium Priority Issues`.
*   **Details:**
    *   **Switch Node:** Routes data based on the `overallRiskScore` relative to the thresholds defined in Block 2.1.
    *   **Slack Nodes:** Sends formatted messages using Block Kit-style text, including a list of "Critical Actions Required" generated by the AI.

### 4. Reproducing the Workflow from Scratch

1.  **Environment Setup:**
    *   Create a new workflow and add a **Schedule Trigger** set to every 6 hours.
    *   Add a **Set** node (`Workflow Configuration`) to define your API URLs and risk thresholds (Critical: 80, High: 60, Medium: 40).
2.  **Data Fetching:**
    *   Connect two **HTTP Request** nodes to fetch your Design and Operational JSON data.
    *   Connect them to a **Set** node (`Merge Engineering Data`) using expressions to map both inputs into a single JSON object.
3.  **AI Layer (The Multi-Agent Cluster):**
    *   **Agents:** Create three **AI Agent** nodes. Configure each with a unique System Message (Design, Safety, Maintenance).
    *   **Model:** Attach an **Anthropic Chat Model** node to each agent (select `Claude 3.5 Sonnet`). Provide Anthropic API credentials.
    *   **Tools:** Create three **AI Agent Tool** nodes. Inside these tools, define specialized tasks (Compliance, Resource, Testing). Connect these tools to the `Tools` input of all three main agents.
    *   **Parsers:** Attach **Structured Output Parser** nodes to every agent/tool to ensure the AI returns valid JSON matching the schema (riskScore, issues, actions).
4.  **Integration & Scoring:**
    *   Add a **Merge** node set to "Combine" mode with 3 inputs to wait for all agent outputs.
    *   Add a **Code** node. Paste the JS logic provided in the analysis to calculate the weighted score based on the three agent outputs.
5.  **Output Logic:**
    *   Add a **Switch** node. Set rules to check the `overallRiskScore` against your configuration thresholds.
    *   Connect **Slack** nodes to the Critical and High outputs. Configure OAuth2 for your Slack Workspace and use expressions to build the alert message.
    *   Connect a **Set** node to the Medium output for internal logging.
