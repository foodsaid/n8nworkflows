# Summarize sales calls and update CRM using WayinVideo and GPT-4o-mini

# 1. Workflow Overview

This workflow collects a sales call recording URL and related deal metadata from a salesperson, sends the recording to WayinVideo for transcription, repeatedly checks until the transcript is available, and then uses GPT-4o-mini to generate a structured CRM-style summary report.

Its main use case is post-call sales ops automation: transforming raw call recordings into actionable CRM notes without requiring manual review. The workflow is designed for teams that want fast call summaries, objections, buying signals, next steps, and deal status classification from Zoom or Google Meet recordings.

The workflow is organized into four logical blocks:

## 1.1 Input Reception

A form is exposed to sales users so they can submit:
- the call recording URL,
- client/company name,
- salesperson name,
- product or service discussed.

This form is the entry point for the workflow.

## 1.2 Transcription Submission and Polling

The submitted recording URL is sent to WayinVideo’s transcript API. Since transcription is asynchronous, the workflow waits 45 seconds and then requests the transcript result.

## 1.3 Transcript Readiness Check and Retry Loop

An IF node checks whether the returned transcript array is non-empty. If the transcript is still unavailable, the workflow loops back to the Wait node and polls again.

## 1.4 AI Report Generation

Once the transcript is available, an AI Agent node receives the transcript and call metadata, uses the GPT-4o-mini chat model, and generates a structured CRM report in a strict template format.

## Important design note

Although the sticky notes state that the final report is appended to a Google Doc, the provided JSON does **not** include any Google Docs node or any persistence/output node after AI generation. In the current workflow, execution ends after the CRM report is generated by the AI Agent.

    2. `Client / Company Name`

## Block 2 — Transcription Submission and Polling

### Overview

This block sends the recording URL to WayinVideo’s transcription API and then pauses execution before polling for results. It handles the asynchronous nature of speech-to-text processing.

### Nodes Involved

- `2. WayinVideo — Submit Transcript`
- `3. Wait — 45 Seconds`
- `4. WayinVideo — Get Transcript`

### Node Details

#### 2. WayinVideo — Submit Transcript

- **Type and technical role:** `n8n-nodes-base.httpRequest`  
  Sends a POST request to WayinVideo to create a transcription job.

- **Configuration choices:**
  - Method: `POST`
  - URL: `https://wayinvideo-api.wayin.ai/api/v2/transcripts`
  - Sends JSON body:
    - `video_url`: from the form field
    - `target_lang`: `"en"`
  - Sends headers:
    - `Authorization: Bearer YOUR_TOKEN_HERE`
    - `x-wayinvideo-api-version: v2`
    - `Content-Type: application/json`

- **Key expressions or variables used:**
  - JSON body expression:
    - `{{ $json['Call Recording URL'] }}`
  - Expected response value used later:
    - `data.id` from the API response

- **Input and output connections:**
  - Input: `1. Form — Call Recording + Details`
  - Output: `3. Wait — 45 Seconds`

- **Version-specific requirements:**
  - Uses `typeVersion: 4.2`
  - Requires the HTTP Request node version that supports JSON body and header parameter configuration as used here.

- **Edge cases or potential failure types:**
  - Invalid or expired API token returns auth errors.
  - Unsupported video URL or inaccessible file may return 4xx errors.
  - API downtime or network issues may return 5xx or timeout errors.
  - If the response structure does not include `data.id`, the next polling node will fail.

- **Sub-workflow reference:**
  - None.

---

#### 3. Wait — 45 Seconds

- **Type and technical role:** `n8n-nodes-base.wait`  
  Delays execution before the first transcript status check and during retries.

- **Configuration choices:**
  - Wait amount: `45` seconds

- **Key expressions or variables used:**
  - None

- **Input and output connections:**
  - Input:
    - `2. WayinVideo — Submit Transcript`
    - `5. IF — Transcript Ready?1` false branch
  - Output:
    - `4. WayinVideo — Get Transcript`

- **Version-specific requirements:**
  - Uses `typeVersion: 1.1`

- **Edge cases or potential failure types:**
  - Long-running workflows may accumulate many waiting executions if transcript jobs are slow.
  - If used in high volume, this can increase execution storage and queue load.
  - It does not include any max retry control.

- **Sub-workflow reference:**
  - None.

---

#### 4. WayinVideo — Get Transcript

- **Type and technical role:** `n8n-nodes-base.httpRequest`  
  Polls WayinVideo for the transcription result using the transcript job ID returned by the submit call.

- **Configuration choices:**
  - Method: default GET
  - URL built dynamically:
    - `https://wayinvideo-api.wayin.ai/api/v2/transcripts/results/{{ $('2. WayinVideo — Submit Transcript').item.json.data.id }}`
  - Sends headers:
    - `Authorization: Bearer YOUR_TOKEN_HERE`
    - `x-wayinvideo-api-version: v2`
    - `Content-Type: application/json`

- **Key expressions or variables used:**
  - Job ID expression:
    - `$('2. WayinVideo — Submit Transcript').item.json.data.id`

- **Input and output connections:**
  - Input: `3. Wait — 45 Seconds`
  - Output: `5. IF — Transcript Ready?1`

- **Version-specific requirements:**
  - Uses `typeVersion: 4.2`

- **Edge cases or potential failure types:**
  - If the previous node failed or returned no `data.id`, this URL expression breaks.
  - If WayinVideo returns a different payload shape than expected, the readiness check may not work.
  - Auth/header errors can prevent polling.
  - A transcript may still be processing, producing an empty or incomplete `data.transcript`.

- **Sub-workflow reference:**
  - None.

## Block 3 — Transcript Readiness Check and Retry Loop

### Overview

This block determines whether the transcript is ready for AI processing. If the transcript array is empty, the workflow loops back and waits another 45 seconds before polling again.

### Nodes Involved

- `5. IF — Transcript Ready?1`

### Node Details

#### 5. IF — Transcript Ready?1

- **Type and technical role:** `n8n-nodes-base.if`  
  Conditional branching node that checks whether a transcript array exists and contains at least one item.

- **Configuration choices:**
  - Condition type: array
  - Operation: `notEmpty`
  - Left value: `{{ $json.data.transcript }}`
  - Strict validation enabled by the node’s condition settings

- **Key expressions or variables used:**
  - `={{ $json.data.transcript }}`

- **Input and output connections:**
  - Input: `4. WayinVideo — Get Transcript`
  - True output: `6. AI Agent — Generate CRM Report1`
  - False output: `3. Wait — 45 Seconds`

- **Version-specific requirements:**
  - Uses `typeVersion: 2.3`
  - Behavior depends on the IF node condition engine version 3 settings.

- **Edge cases or potential failure types:**
  - If `data.transcript` is missing rather than empty, strict validation may cause the check to behave unexpectedly depending on actual runtime data.
  - If the API returns transcript text in another field, the loop never exits.
  - This node creates a potentially infinite loop because there is no retry counter or stop condition.
  - A malformed API response can repeatedly route to the retry branch forever.

- **Sub-workflow reference:**
  - None.

## Block 4 — AI Report Generation

### Overview

After the transcript is confirmed available, this block builds a highly structured prompt containing form metadata and a speaker-labeled transcript. The AI Agent uses GPT-4o-mini to classify and summarize the sales call in CRM-oriented language.

### Nodes Involved

- `OpenAI Chat Model — GPT-4o-mini`
- `6. AI Agent — Generate CRM Report1`

### Node Details

#### OpenAI Chat Model — GPT-4o-mini

- **Type and technical role:** `@n8n/n8n-nodes-langchain.lmChatOpenAi`  
  Provides the language model backend for the AI Agent.

- **Configuration choices:**
  - Model selected from list: `gpt-4o-mini`
  - No additional options configured
  - No built-in tools enabled

- **Key expressions or variables used:**
  - None directly in this node

- **Input and output connections:**
  - Input: none on the main branch
  - AI language model output connected to:
    - `6. AI Agent — Generate CRM Report1`

- **Version-specific requirements:**
  - Uses `typeVersion: 1.3`
  - Requires n8n installation with LangChain/AI nodes enabled.
  - Requires valid OpenAI credentials configured in n8n.

- **Edge cases or potential failure types:**
  - Invalid OpenAI credentials
  - Model access restrictions on the connected OpenAI account
  - Rate limiting or token limit issues for long transcripts
  - Inference latency on large transcript payloads

- **Sub-workflow reference:**
  - None.

#### 6. AI Agent — Generate CRM Report1

- **Type and technical role:** `@n8n/n8n-nodes-langchain.agent`  
  Main LLM orchestration node that receives prompt instructions and transcript data, then generates a structured CRM report.

- **Configuration choices:**
  - Prompt type: `define`
  - Custom prompt instructs the agent to behave as a CRM specialist
  - Explicitly defines speaker roles:
    - Speaker 1 = salesperson
    - Speaker 2 = client
  - Injects call metadata from the form
  - Injects formatted transcript lines from the WayinVideo result
  - Enforces an exact output structure with sections for:
    - Date
    - Salesperson
    - Client
    - Product discussed
    - Recording URL
    - Call summary
    - Action items
    - Deal status with reason
    - Client objections
    - Buying signals
    - Pricing discussed
    - Next follow-up
    - Overall call sentiment

- **Key expressions or variables used:**
  - Salesperson:
    - `{{ $('1. Form — Call Recording + Details').item.json['Salesperson Name'] }}`
  - Client/company:
    - `{{ $('1. Form — Call Recording + Details').item.json['Client / Company Name'] }}`
  - Product/service:
    - `{{ $('1. Form — Call Recording + Details').item.json['Product / Service Discussed'] }}`
  - Recording URL:
    - `{{ $('1. Form — Call Recording + Details').item.json['Call Recording URL'] }}`
  - Transcript formatter:
    - `{{ $('4. WayinVideo — Get Transcript').item.json.data.transcript.map(s => '[' + s.speaker + ' | ' + Math.floor(s.start/1000) + 's] ' + s.text).join('\n') }}`

- **Input and output connections:**
  - Main input: `5. IF — Transcript Ready?1` true branch
  - AI model input: `OpenAI Chat Model — GPT-4o-mini`
  - Main output: none connected in the provided workflow

- **Version-specific requirements:**
  - Uses `typeVersion: 3.1`
  - Requires compatible LangChain Agent support in the installed n8n version.

- **Edge cases or potential failure types:**
  - If `data.transcript` contains unexpected objects or missing fields (`speaker`, `start`, `text`), the transcript formatting expression may fail.
  - Long transcripts may exceed model context limits or cause truncation.
  - The assumption that Speaker 1 is always the salesperson and Speaker 2 is always the client may be wrong for some calls.
  - The output is not persisted because no downstream storage node exists.
  - If the transcript contains poor diarization, the CRM report quality may degrade significantly.

- **Sub-workflow reference:**
  - None.

# 4. Reproducing the Workflow from Scratch

Below is the exact manual rebuild process based on the provided workflow JSON.

1. **Create a new workflow in n8n**
   - Give it the title: `Summarize sales calls and update CRM using WayinVideo and GPT-4o-mini`

2. **Add a Form Trigger node**
   - Node type: `Form Trigger`
   - Name it: `1. Form — Call Recording + Details`
   - Set form title to: `📞 Sales Call Auto-Summarizer & CRM Updater`
   - Set form description to:  
     `Paste your sales call recording URL — AI will extract CRM notes, action items, deal status and next steps automatically.`
   - Add 4 required fields:
     1. `Call Recording URL`
        - Placeholder: `https://zoom.us/rec/xxxxxxx or Google Meet recording link`
     2. `Client / Company Name`
        - Placeholder: `e.g. Rahul Sharma / ABC Digital Solutions`
     3. `Salesperson Name`
        - Placeholder: `e.g. Deepak Verma`
     4. `Product / Service Discussed`
        - Placeholder: `e.g. SEO Package, GEO Services, Social Media Management`

3. **Add an HTTP Request node to submit the transcript job**
   - Node type: `HTTP Request`
   - Name it: `2. WayinVideo — Submit Transcript`
   - Method: `POST`
   - URL: `https://wayinvideo-api.wayin.ai/api/v2/transcripts`
   - Enable sending headers
   - Add headers:
     - `Authorization` = `Bearer YOUR_TOKEN_HERE`
     - `x-wayinvideo-api-version` = `v2`
     - `Content-Type` = `application/json`
   - Enable sending body
   - Set body type to JSON
   - Use this JSON body:
     ```json
     {
       "video_url": "{{ $json['Call Recording URL'] }}",
       "target_lang": "en"
     }
     ```
   - In n8n UI, this should be configured as an expression-enabled JSON body.
   - Replace `YOUR_TOKEN_HERE` with the actual WayinVideo bearer token.

4. **Connect the Form Trigger to the submit node**
   - `1. Form — Call Recording + Details` → `2. WayinVideo — Submit Transcript`

5. **Add a Wait node**
   - Node type: `Wait`
   - Name it: `3. Wait — 45 Seconds`
   - Set wait amount to `45` seconds

6. **Connect the submit node to the Wait node**
   - `2. WayinVideo — Submit Transcript` → `3. Wait — 45 Seconds`

7. **Add a second HTTP Request node to retrieve transcript results**
   - Node type: `HTTP Request`
   - Name it: `4. WayinVideo — Get Transcript`
   - Method: `GET`
   - URL as expression:
     ```javascript
     =https://wayinvideo-api.wayin.ai/api/v2/transcripts/results/{{ $('2. WayinVideo — Submit Transcript').item.json.data.id }}
     ```
   - Enable sending headers
   - Add headers:
     - `Authorization` = `Bearer YOUR_TOKEN_HERE`
     - `x-wayinvideo-api-version` = `v2`
     - `Content-Type` = `application/json`
   - Use the same WayinVideo token as above.

8. **Connect the Wait node to the transcript retrieval node**
   - `3. Wait — 45 Seconds` → `4. WayinVideo — Get Transcript`

9. **Add an IF node for transcript readiness**
   - Node type: `IF`
   - Name it: `5. IF — Transcript Ready?1`
   - Create one condition:
     - Left value: `{{ $json.data.transcript }}`
     - Operator type: `Array`
     - Operation: `is not empty` / `notEmpty`
   - Keep the condition combinator as `and`

10. **Connect transcript retrieval to the IF node**
    - `4. WayinVideo — Get Transcript` → `5. IF — Transcript Ready?1`

11. **Create the retry loop**
    - Connect the **false** output of `5. IF — Transcript Ready?1` back to:
      - `3. Wait — 45 Seconds`
    - This makes the workflow poll every 45 seconds until a non-empty transcript is returned.

12. **Add an OpenAI Chat Model node**
    - Node type: `OpenAI Chat Model`
    - Name it: `OpenAI Chat Model — GPT-4o-mini`
    - Select model: `gpt-4o-mini`
    - Credentials:
      - Connect valid OpenAI credentials in n8n
    - Leave other options at default unless needed

13. **Add an AI Agent node**
    - Node type: `AI Agent`
    - Name it: `6. AI Agent — Generate CRM Report1`
    - Prompt type: `Define`
    - Paste the following prompt content into the main prompt field:

      ```text
      You are a CRM specialist. Your job is to analyze a sales call transcript and extract structured CRM data.

      IMPORTANT: Speaker 1 is the salesperson. Speaker 2 is the client.

      Call Details:
      - Salesperson: {{ $('1. Form — Call Recording + Details').item.json['Salesperson Name'] }}
      - Client / Company: {{ $('1. Form — Call Recording + Details').item.json['Client / Company Name'] }}
      - Product / Service Discussed: {{ $('1. Form — Call Recording + Details').item.json['Product / Service Discussed'] }}
      - Recording URL: {{ $('1. Form — Call Recording + Details').item.json['Call Recording URL'] }}

      Full Call Transcript:
      {{ $('4. WayinVideo — Get Transcript').item.json.data.transcript.map(s => '[' + s.speaker + ' | ' + Math.floor(s.start/1000) + 's] ' + s.text).join('\n') }}

      Extract the CRM data and write a structured report using this exact format. Do not skip any section.

      === SALES CALL CRM REPORT ===

      DATE: [Today's date]
      SALESPERSON: [Salesperson name from above]
      CLIENT: [Client name from above]
      PRODUCT DISCUSSED: [Product name from above]
      CALL RECORDING: [Recording URL from above]

      ---

      CALL SUMMARY:
      [Write 3 to 4 sentences about what was discussed in the call. Include the main topic, what the client asked, and how the conversation ended.]

      ---

      ACTION ITEMS:
      [List every task the salesperson must do after this call. One action per line starting with a dash.]
      - 
      - 
      - 

      ---

      DEAL STATUS:
      [Choose exactly one: Hot Lead / Warm Lead / Cold Lead / Negotiating / Closed Won / Closed Lost]
      Reason: [Write 1 sentence explaining why you chose this status]

      ---

      CLIENT OBJECTIONS:
      [List every concern or problem the client raised. One per line starting with a dash. If none, write: None raised.]
      - 

      ---

      BUYING SIGNALS:
      [List every positive statement or sign of interest from the client. One per line starting with a dash. If none, write: None detected.]
      - 

      ---

      PRICING DISCUSSED:
      [Write any price, budget, or discount mentioned during the call. If nothing was discussed, write: Not discussed.]

      ---

      NEXT FOLLOW-UP:
      [Write the exact date or timeline mentioned for the next step. If nothing was mentioned, write: Not specified.]

      ---

      OVERALL CALL SENTIMENT: [Choose one: Positive / Neutral / Negative]

      === END OF REPORT ===
      ```

14. **Connect the OpenAI Chat Model to the AI Agent**
    - Use the AI language model connection:
      - `OpenAI Chat Model — GPT-4o-mini` → `6. AI Agent — Generate CRM Report1`

15. **Connect the IF true branch to the AI Agent**
    - `5. IF — Transcript Ready?1` true output → `6. AI Agent — Generate CRM Report1`

16. **Leave the AI Agent output unconnected**
    - This matches the provided JSON exactly.
    - Important: this means the report is generated but not stored anywhere.

17. **Configure credentials**
    - **WayinVideo**
      - No dedicated credential object is used in the provided workflow.
      - Token is manually inserted into HTTP headers in both WayinVideo HTTP nodes.
    - **OpenAI**
      - Connect an OpenAI credential to the Chat Model node.

18. **Optionally add the sticky notes for documentation**
    - These are informational only and do not affect execution.
    - Add note content matching the workflow if you want visual parity.

19. **Test the workflow**
    - Submit a publicly accessible Zoom or Google Meet recording URL.
    - Verify `2. WayinVideo — Submit Transcript` returns a `data.id`.
    - Verify repeated polling eventually returns a populated `data.transcript`.
    - Confirm `6. AI Agent — Generate CRM Report1` produces the structured report.

20. **Recommended hardening before production**
    - Add a retry counter using a Set node plus IF node to stop after a maximum number of loops.
    - Add an error branch for failed WayinVideo requests.
    - Add a storage destination, since the current workflow does not save the report.

## Sub-workflow setup

- No sub-workflow nodes are present.
- No workflow invocation nodes are present.
- No child workflow input/output contracts are required.

## Important mismatch to be aware of

The notes in the workflow mention:
- replacing a Google Doc URL or ID,
- connecting Google Docs OAuth2,
- appending the report to a shared Google Doc.

However, the actual JSON contains **no Google Docs node**. If you want the described behavior, you must add an additional Google Docs node after the AI Agent to append the generated report.
