# Monitor news sentiment trends with GPT-4o-mini, MrScraper, Slack and Notion

This document provides a comprehensive technical breakdown of the **Sentiment & Trend Radar Monitoring** workflow. This system is designed to automate news monitoring, sentiment analysis, and reporting using a combination of web scraping, LLM-based analysis, and multi-channel notifications.

---

### 1. Workflow Overview

The workflow monitors specific news sources daily to detect sentiment trends and identify articles requiring immediate attention. It uses **MrScraper** for data extraction, **OpenAI (GPT-4o-mini)** for qualitative analysis, and routes the findings to **Notion** for archival and **Slack** for real-time alerting.

**Logical Blocks:**
*   **1.1 Source & URL Discovery:** Triggers the process, fetches target websites from Google Sheets, and identifies article URLs.
*   **1.2 Content Extraction & Filtering:** Scrapes full article text and filters out irrelevant or low-quality content based on keywords and word count.
*   **1.3 AI Intelligence Layer:** Analyzes the article's sentiment, tone, and urgency using an LLM.
*   **1.4 Alerting & Archiving:** Sends immediate Slack alerts for critical items and saves all data to a Notion database.
*   **1.5 Reporting:** Aggregates all daily results into a single "Daily Digest" Slack report.

### 2. Block-by-Block Analysis

#### 2.1 Source & URL Discovery
**Overview:** This block initializes the workflow and discovers links to analyze.
*   **Nodes Involved:** `Schedule Trigger`, `Get Target News Website`, `Loop Each News Source`, `MrScraper - Discover Article URLs`, `Extract URLs`.
*   **Node Details:**
    *   **Schedule Trigger:** Fires every 24 hours at 08:00 AM.
    *   **Get Target News Website (Google Sheets):** Fetches the list of domains or search URLs (e.g., Google News searches) to monitor.
    *   **MrScraper - Discover Article URLs:** Uses the `mapAgent` operation to identify potential article links.
    *   **Extract URLs (Code):** A JavaScript node that parses the MrScraper JSON response to create a clean list of URLs for the next loop.
*   **Edge Cases:** Empty Google Sheet, MrScraper timeout, or no URLs discovered on the source page.

#### 2.2 Content Extraction & Filtering
**Overview:** Processes discovered URLs individually to extract and validate content.
*   **Nodes Involved:** `Loop Each Article URL`, `MrScraper - Extract Article Content`, `Pick Best Content + Filter Irrelevant`, `Keep Only Relevant Articles`.
*   **Node Details:**
    *   **MrScraper - Extract Article Content:** Uses `generalAgent` to pull full text, author, and date.
    *   **Pick Best Content (Code):** Filters articles. Hardcoded keywords in this node include `"Hukum"`, `"iran"`, and `"education"`. It also rejects articles with `< 50 words`.
    *   **Keep Only Relevant Articles (IF):** Checks the `skip` flag generated by the previous code node.
*   **Edge Cases:** Paywalled content, 404 errors on article pages, or irrelevant content that doesn't meet keyword criteria.

#### 2.3 AI Intelligence Layer
**Overview:** Performs the core qualitative analysis of the news.
*   **Nodes Involved:** `Sentiment & Summary AI Agent`, `OpenAI Chat Model`, `Parse AI Response & Format Output`.
*   **Node Details:**
    *   **OpenAI Chat Model:** Configured with `gpt-4o-mini` for cost-efficiency.
    *   **Sentiment & Summary AI Agent:** A LangChain node with a strict system prompt. It mandates a JSON response containing `sentiment_score` (-1 to 1), `tone`, and `action_required`.
    *   **Parse AI Response (Code):** Strips markdown formatting (backticks) from the AI's response and calculates emojis for Slack based on sentiment.
*   **Edge Cases:** AI hallucination, JSON parsing errors (handled by a `try/catch` in the code node), or OpenAI API rate limits.

#### 2.4 Alerting, Archiving & Reporting
**Overview:** Distributes findings to stakeholders and saves data for historical tracking.
*   **Nodes Involved:** `Needs Urgent Alert?`, `Slack - Urgent Alert`, `Notion - Save Article Analysis`, `Build Daily Digest Summary`, `Slack - Daily Digest`.
*   **Node Details:**
    *   **Notion Node:** Maps AI analysis to database properties like `Sentiment Score`, `Tone`, and `Action Reason`.
    *   **Build Daily Digest (Code):** Uses `$input.all()` to aggregate results from the entire batch. Calculates the average sentiment score and identifies "Top Articles."
    *   **Slack Nodes:** One for immediate "🚨 URGENT" alerts and one for the "📰 Daily Sentiment Report."
*   **Edge Cases:** Slack API downtime, Notion database ID mismatch, or 0 articles processed (handled by a fallback in the Digest node).

### 4. Reproducing the Workflow from Scratch

1.  **Google Sheets Setup:** Create a sheet with a column for "URL". Populate it with news search URLs (e.g., Google News or RSS links).
2.  **MrScraper Configuration:**
    *   Create a **Map Agent** (for URL lists) and a **General Agent** (for full article text).
    *   Note the `scraperId` for both.
3.  **Notion Setup:** Create a database with properties: `Sentiment` (Select), `Sentiment Score` (Number), `Tone` (Select), `Source URL` (URL), and `Action Required` (Checkbox).
4.  **n8n Node Setup:**
    *   **Trigger:** Add a `Schedule Trigger` (8:00 AM).
    *   **Retrieval:** Connect `Google Sheets` to read your URLs.
    *   **Nesting Loops:** Create an outer loop (`Split in Batches`) for news sources and an inner loop for individual articles.
    *   **Scraping:** Add two `MrScraper` nodes. The first uses the Discovery ID, the second uses the Extraction ID.
    *   **Logic (Filtering):** Add a `Code` node after extraction. Use `item.content.split(' ').length` to filter out short texts and check for keywords using `.includes()`.
    *   **Intelligence:** Add the `AI Agent` (LangChain). Inside the `OpenAI Chat Model`, set the model to `gpt-4o-mini`. 
    *   **Prompting:** In the AI Agent, use a prompt that demands a raw JSON object (no markdown) to avoid parsing errors.
    *   **Branching:** Use an `IF` node to check `action_required`. If true, route to a `Slack` node.
    *   **Aggregation:** Connect the "Done" output of the **News Source Loop** to a `Code` node. Use `$input.all()` to gather every processed item and format a single string summary.
5.  **Connections:** Ensure the `Notion` node connects back to the `Loop Each Article URL` node to continue the cycle until all articles are processed.
