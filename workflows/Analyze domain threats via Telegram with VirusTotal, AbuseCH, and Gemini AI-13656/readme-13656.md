# Workflow Reference: Analyze domain threats via Telegram with VirusTotal, AbuseCH, and Gemini AI

This workflow implements a professional cybersecurity automated assistant accessible via Telegram. It allows users to submit a domain for real-time threat intelligence gathering across multiple industry-standard databases, followed by an AI-driven analysis to provide a risk assessment and actionable recommendations.

---

### 1. Workflow Overview

The workflow follows a linear logic of data ingestion, parallel enrichment, and consolidated synthesis:

*   **1.1 Input Reception:** A Telegram bot waits for a message containing a domain name.
*   **1.2 Threat Intelligence Gathering:** The workflow simultaneously queries VirusTotal, Abuse.CH URLHaus, and Abuse.CH ThreatFox to gather reputation data and malicious activity history.
*   **1.3 Data Normalization:** Raw JSON responses from the intelligence engines are tagged with their source and stringified to prepare them for AI consumption.
*   **1.4 AI Processing:** A Google Gemini model (acting as a SOC Analyst) analyzes the consolidated findings to generate a human-readable report.
*   **1.5 Response Delivery:** The final report is sent back to the original Telegram chat with proper HTML formatting.

---

### 2. Block-by-Block Analysis

#### 2.1 Entry & Trigger
This block initiates the process upon receiving a user interaction.

*   **Node Involved:** Telegram Trigger.
*   **Details:**
    *   **Type:** Telegram Trigger.
    *   **Configuration:** Listens for `message` updates.
    *   **Key Security Note:** It is highly recommended to use the "Chat IDs" additional field to restrict the bot to specific authorized users or groups.
    *   **Output:** The message text (the domain) and user metadata.

#### 2.2 Threat Intelligence Engines
Three parallel requests fetch data from external security APIs.

*   **Nodes Involved:** VirusTotal HTTP Request, Abuse.CH_URLHaus request, Abuse.CH_ThreatFox request.
*   **Node Details:**
    *   **VirusTotal:** A `GET` request to `https://www.virustotal.com/api/v3/domains/{{ domain }}`. Requires a VirusTotal API key. Handles potential 404s via "Continue Regular Output" to prevent workflow crashes if a domain is unknown to VT.
    *   **Abuse.CH URLHaus:** A `POST` request to `https://urlhaus-api.abuse.ch/v1/host/` using `form-urlencoded` body to check for known malicious URLs.
    *   **Abuse.CH ThreatFox:** A `POST` request to `https://threatfox-api.abuse.ch/api/v1/` using a JSON body (`search_ioc`) to identify Indicators of Compromise (IOCs).

#### 2.3 Data Structuring & Normalization
This block transforms disparate API responses into a uniform format for the AI.

*   **Nodes Involved:** Edit Field - VirusTotal, Edit Field - URLHaus, Edit Field - ThreatFox.
*   **Details:**
    *   **Type:** Set (Edit Fields).
    *   **Configuration:** Each node adds a `sourceNode` string (e.g., "VirusTotal") and a `DataSummary` field containing the stringified JSON of the previous node's output.
    *   **Logic:** This ensures the AI receives a clear map of which data comes from which security engine.

#### 2.4 Consolidation
Gathers the three parallel outputs into a single package.

*   **Nodes Involved:** Merge, Aggregate.
*   **Details:**
    *   **Merge:** Combines the three branches into a single stream of 3 items.
    *   **Aggregate:** Uses the "Aggregate All Item Data" option to merge those 3 items into a single array, providing the AI with one comprehensive context window.

#### 2.5 AI Synthesis (The Analyst)
The core intelligence logic that interprets the security data.

*   **Node Involved:** Message a model (Google Gemini).
*   **Details:**
    *   **Model:** Gemini 1.5 Flash (or Gemini 3 Flash Preview).
    *   **System Prompt:** Defines the persona as an "Experienced Cybersecurity Analyst Assistant." It mandates a neutral, evidence-based tone, defanging of domains (replacing `.` with `[.]`), and specific formatting constraints for Telegram.
    *   **User Prompt:** Passes the aggregated data and specific instructions on how to interpret VirusTotal stats, URLHaus blacklists, and ThreatFox tags.

#### 2.6 Report Delivery
The final stage of the workflow.

*   **Node Involved:** Send a text message (Telegram).
*   **Details:**
    *   **Configuration:** Sends the AI-generated text back to the `chat.id` captured in the trigger.
    *   **Formatting:** Enables `HTML` parse mode to support the bold, italic, and underline tags generated by the AI.

### 4. Reproducing the Workflow from Scratch

1.  **Trigger Setup:** Create a **Telegram Trigger** node. Connect your Telegram Bot Token. In "Additional Fields," add "Chat IDs" to secure the bot.
2.  **Intelligence Branching:**
    *   Add three **HTTP Request** nodes connected to the Trigger.
    *   **VT:** Method GET, URL `https://www.virustotal.com/api/v3/domains/{{ $json.message.text }}`. Set Authentication to Header (X-Apikey).
    *   **URLHaus:** Method POST, URL `https://urlhaus-api.abuse.ch/v1/host/`. Body: `host={{ $json.message.text }}`.
    *   **ThreatFox:** Method POST, URL `https://threatfox-api.abuse.ch/api/v1/`. Body JSON: `{"query": "search_ioc", "search_term": "{{ $json.message.text }}"}`.
3.  **Data Labeling:**
    *   Attach a **Set** node after each HTTP request. 
    *   Create a string field `sourceNode` (unique to each branch).
    *   Create a string field `DataSummary` with the expression `{{ JSON.stringify($json) }}`.
4.  **Consolidation:**
    *   Add a **Merge** node with "Number of Inputs" set to 3. Connect all three Set nodes.
    *   Add an **Aggregate** node. Select "Aggregate All Item Data."
5.  **AI Integration:**
    *   Add a **Google Gemini** node. Use the "Message a model" action.
    *   Paste the System Prompt defining the "Cybersecurity Analyst" persona.
    *   In the User Prompt, reference the data using: `{{ $json.data[0]['DataSummary'] }}`.
6.  **Final Output:**
    *   Add a **Telegram** node (Action: Send Text Message).
    *   Set the Chat ID to `{{ $('Telegram Trigger').item.json.message.chat.id }}`.
    *   Set Text to the Gemini output: `{{ $json.candidates[0].content.parts[0].text }}`.
    *   Ensure Parse Mode is set to **HTML**.
